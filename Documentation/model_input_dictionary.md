# Model Input Dictionary

**Created:** 2026-06-11
**Purpose:** Every piece of data the production engine consumes — what it is, where the code reads it from, which source channel it ultimately comes from, the fallback chain, and the hard-coded constants. This is the "schema" side of the provenance catalogue; the per-plan "instance" side is `Documentation/provenance_register.csv` (generated by `Documentation/provenance_scan.py`).

**Authority:** derived by reading the production code (`Code/python/fast/Main_PensionModel.py`, `Code/python/functions_cf_model.py` helpers, `Code/python/asset_simulation.py`), not from prior documentation. Code citations are to the fast runner unless noted.

**Source channels** (the vocabulary used throughout):
| Channel | File | Who made it |
|---|---|---|
| `plan_workbook` | `Data/Plans/States/[PLAN]/[PLAN]_2017.xlsx` | Brookings extraction from AVs (FY2017) |
| `default_assumptions` | `Data/Common/states/default_assumptions.xlsx` | Brookings-inherited generic actuarial tables |
| `ppd_panel` | `Data/Common/states/ppd-data-latest.xlsx` (row `ppd_id` × `fy`) | CRR Boston College digest of AV/CAFRs |
| `legacy_ppd_csv` | `Data/Common/states/PPD_planlevel_main.csv` (`planid=[PLAN]_2017`) and `PPD_planlevel_main_updated.csv` (`planid=[PLAN]_2022`) | Brookings-era per-plan scalar files |
| `planchanges` | `Data/Common/states/planchanges_main_2022_clean.xlsx`, sheet `in`, key `[PLAN]_[year]` | hand-curated tier rules (2022 update, 2024) |
| `constant` | hard-coded in the engine | model assumption |

## 1. Distribution inputs (the 9 workbook sheets)

`availableData` is the per-plan 9-boolean vector hard-coded in the runner (`AVAILABLE_DATA` dict, fast runner ~line 36): `[ageservice, retdist, wagerel, mortality, wagegrowth, withdrawal, retirement, refund, disability]`. `False` → the `default_assumptions` sheet of the same name is used instead.

| # | Sheet | Model object | What it is | Read as | Scaling applied | Fallback when flag=False |
|---|---|---|---|---|---|---|
| 1 | `ageservice` | `active` (via `LinearFill`) | Active-member shares by age×service bucket | `B2:L12` (11×11 fractions) | × `ppd_panel.actives_tot` | default `ageservice` |
| 2 | `retdist` | `RetirementNumber`, `RetirementBenefit` | Current-retiree age distribution + benefit relativities | col `B` (counts share), col `F` (benefit rel.), 16 rows | × `beneficiaries_tot`; × `BeneficiaryBenefit_avg`×1000 | default `retdist` |
| 3 | `wagerel` | `BaseWage_2d` (via `ConstantFill`) | Relative wage by age×service | `B2:L12` | × `ActiveSalary_avg`×1000 | default `wagerel` |
| 4 | `mortality` | `MortalityTable` | Mortality rates, male (`B:D`) + female (`F:H`), 4 age points each | 2 blocks, 4 rows | blended with `pctmale` via `mort_table_fast` | default `mortality` |
| 5 | `wagegrowth` | — | **GHOST SHEET: never read by the engine.** Wage growth is the PPD scalar chain (§3). Sheet exists in workbooks for provenance only. | — | — | — |
| 6 | `withdrawal` | `SeparationRate` | Separation/turnover rates, male (`A:L`) + female (`N:Y`) | 12×12, /100 | blended with `pctmale` | default `withdrawal` (single block) |
| 7 | `retirement` | `RetirementRate` | Retirement propensity by age×service | `Q:AA` 11 rows, /100, negatives→0 | — | default `retirement` (`B:L`) |
| 8 | `refund` | `RefundRate` | Refund rates on separation, male (`B:L`) + female (`O:Y`) | 11×11 | blended with `pctmale` | default `refund` (single block) |
| 9 | `disability` | — | **GHOST SHEET: never read.** Disability payout is the constant `DisabilityPayoutRate = 0.025` (2.5% of payroll). | — | — | — |

Note the **sex-blending asymmetry**: plan-specific mortality/withdrawal/refund come as male+female blocks blended with `pctmale`; the defaults are single pre-blended blocks. The Brookings `corresponding CSV matrices` layer holds the even-more-granular precursors (`mortalityact`/`mortalityret`, `withdrawalf`/`withdrawalm`, `retirementn`/`retiremento`, `retbenrel`) that the workbooks condensed.

## 2. Demographic scalars (legacy_ppd_csv only — NOT in the PPD panel)

| Variable | Meaning | Used in |
|---|---|---|
| `pctmale` | share of members male | blending all sex-split sheets |
| `pctmrg` (`pct_mrg`) | share married | death-benefit (widow) calc |
| `reduct` (`widow_reduct`) | survivor-benefit reduction factor | death-benefit calc |
| `inactive_adj` | flag/multiplier choosing the inactive-scaling rule | §3 inactive chain |

These come from `PPD_planlevel_main_updated.csv` keyed `[PLAN]_2022`, which **only contains the 40 state plans** — a city integration blocker (see `city_data_audit.md` §6.1).

## 3. PPD-panel scalars and their fallback chains

Read from the `ppd_id` × `fy=plan_year` row of `ppd-data-latest.xlsx`:

| Model variable | Source expression | Fallback chain (from `functions_cf_model.py` helpers) |
|---|---|---|
| `Assets[0]` | `ActAssets_GASB` × 1000 | none (required) |
| `discountrate` | `InvestmentReturnAssumption_GASB` | none; `--discount-override` CLI replaces it (scenario lever) |
| `EmployeeContributionRate` | `contrib_EE_regular / payroll` | if NaN (plans in `CONTRIB_RATE_NA_CHECK`): `contrib_EE_regular×1000 / Σ(active×BaseWage)` |
| `EmployerContributionRate` | `contrib_ER_regular / payroll` | same recomputation |
| `WageGrowth` | `get_wage_growth_assumption` | `PayrollGrowthAssumption` → `WageInflation` → legacy `wage_inf` ([PLAN]_2017) → `InflationAssumption_GASB` → legacy `inflation` |
| `Inflation` | `get_inflation_assumption` | `InflationAssumption_GASB` → legacy `inflation` |
| inactive scaling | `scale_inactive_members` | if `inactive_adj==1`: `InactiveVestedMembers` → legacy `inactive`; else `actives_tot × inactive_adj` |
| `actives_tot` | scales `ageservice` shares to counts | none |
| `beneficiaries_tot`, `BeneficiaryBenefit_avg` | scale `retdist` | none |
| `ActiveSalary_avg` | scales `wagerel` | none |
| `CAFR_AAL` (validation only) | `ActLiabilities_GASB` × 1000 | none |

**Vintage note (the 2022 layering):** in the canonical 062026 run these scalars come from the **fy2022** PPD row, while the §1 distributions remain the **FY2017 Brookings extractions** — the documented "hybrid" (see `year_version_audit.md`). The legacy-csv fallbacks reach back to 2017 values by design.

## 4. Tier rules (`planchanges`)

Sheet `in`, row `planid=[PLAN]_[plan_year]`, fields per tier i=1..6: `startdate{i}`, `benefitfactor{i}`, `vesting{i}`, `maxsal{i}` (−100 → cap 100), `yrsal{i}` (salary-averaging window), `nr{i}`/`er{i}` (normal/early retirement age), `cola{i}`. Duplicate tier rows are collapsed (`drop_duplicates` → `num_tiers`); tier service boundaries from `round((plan_start − startdate)/52.25 weeks)`. The workbook also carries `our_cola{i}`/`type{i}`/`compounded{i}`/`eecont{i}`/`ercont{i}` — **`eecont`/`ercont` are NOT consumed** (one plan-level contribution rate is used; documented future extension). Provenance: hand-curated during the 2022 update (cleaned 2024), originally derived from Brookings' `planchanges_main.xlsx`.

## 5. Asset-stage inputs (`asset_simulation.py`)

| Input | Source | Notes |
|---|---|---|
| Liabilities/cashflows/NormalCost/contrib rates | the detAL pkl | pass-through of §1–§4 results |
| `AssetShare` (2-asset) | `ppd_panel`: risky = `EQTotal_Actl + PETotal_Actl + HFTotal_Actl + RETotal_Actl + COMDTotal_Actl + AltMiscTotal_Actl + OtherTotal_Actl`; bonds = 1 − risky | computed from saved `planinfo` if absent |
| stock return | `(stock_premium + Inflation_p) + stock_vol × Z[t,n]` | `Z` = ONE standardized shock matrix shared by all plans (market seed; canonical = 123) |

## 7. City-track mapping (for the future type-blind registry)

City collection sheets → state inputs: `Age_Serv_Num→ageservice`, `Retirement→retdist`, `Age_Serv_Wage→wagerel`, `Avg_Mort→mortality`, `Wage_Growth→(ghost: wagegrowth)`, `Sep_Rate→withdrawal`, `Ret_Rate→retirement`, `Refund_Rate→refund`, `Inactv_Serv_Num→(inactive scaling input)`. `_tiervars.xlsx` → `planchanges` rows.
